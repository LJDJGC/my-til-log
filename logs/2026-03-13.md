### 💡 学びのポイント
**新規習得：**
- `String.prototype.padStart()`: これまで `if (hours < 10) hours = '0' + hours;` と書いていたゼロ埋め処理を、`String(date.getHours()).padStart(2, '0')` のように1行で宣言的に書ける手法。可読性と保守性が向上する。

**理解深化：**
- **メソッドとプロトタイプ:** class構文で定義したメソッドは、各インスタンス内に生成されるのではなく「プロトタイプ」に格納される。100個のインスタンスを作っても関数自体はメモリ上に1つしか存在せず、メモリ効率が良い。
- **`this` の挙動とアロー関数:** `setInterval` のコールバックとしてメソッドを渡す際、単に `setInterval(this.render, 1000)` とすると `this` が呼び出し元のコンテキストに縛られ、`undefined` になる。これを防ぐため、外側の `this` を引き継ぐアロー関数を使い `setInterval(() => this.render(), 1000)` と記述する重要性を理解した。

**トラブルシューティング：**
- リファクタリング時、`this` の喪失問題と `padStart` の適用を同時に解決。単に「動く」だけでなく「なぜそう書くのか」を言語化してコミットメッセージに残した。

### 🔗 実装コード
- リポジトリ：playground（ローカルコミット）
- 主要コード：
```javascript
class Clock {
  constructor({ template }) {
    this.template = template;
    this.timer = null;
  }

  render() {
    let date = new Date();
    
    // 冗長なif文をpadStartにリファクタリング
    let hours = String(date.getHours()).padStart(2, '0');
    let mins = String(date.getMinutes()).padStart(2, '0');
    let secs = String(date.getSeconds()).padStart(2, '0');

    let output = this.template
      .replace('h', hours)
      .replace('m', mins)
      .replace('s', secs);

    console.log(output);
  }

  stop() {
    clearInterval(this.timer);
  }

  start() {
    this.render();
    // thisの喪失を防ぐためアロー関数を使用
    this.timer = setInterval(() => this.render(), 1000);
  }
}

let clock = new Clock({ template: 'h:m:s' });
clock.start();
```

### 💡 Learning Points
**Newly Acquired:**
- `String.prototype.padStart()`: A technique that allows you to declaratively write zero-padding, previously written as `if (hours < 10) hours = '0' + hours;`, in a single line like `String(date.getHours()).padStart(2, '0')`. Improves readability and maintainability.

**Deepening Understanding:**
- **Methods and Prototypes:** Methods defined using class syntax are stored in a "prototype" rather than being created within each instance. Even with 100 instances, only one instance of the function itself exists in memory, resulting in good memory efficiency.
- **Behavior of `this` and Arrow Functions:** When passing a method as a callback to `setInterval`, simply writing `setInterval(this.render, 1000)` causes `this` to be tied to the caller's context and become `undefined`. I understood the importance of using an arrow function that inherits the outer `this` to prevent this, writing `setInterval(() => this.render(), 1000)`.

**Troubleshooting:**
- During refactoring, I simultaneously resolved the `this` loss issue and the application of `padStart`. I not only made it "work," but also articulated "why it's written that way" and included it in the commit message.
