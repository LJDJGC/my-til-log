### 💡 学びのポイント
**新規習得：**
- **API Route Handler でサードパーティAPIをプロキシするパターン**:
  ブラウザ → `/api/youtube/search?q=...`（自サーバー）→ YouTube Data API（サーバーサイド）
  こうすることで API キーをブラウザに露出させずに済む。
  ```tsx
  // Route Handler（サーバーサイド）
  const apiKey = process.env.YOUTUBE_API_KEY; // ブラウザ非公開
  const url = new URL("https://www.googleapis.com/youtube/v3/search");
  url.searchParams.set("key", apiKey);
  const response = await fetch(url.toString());
  ```
- **`NEXT_PUBLIC_` プレフィックスの意味**:
  これを付けるとビルド時に環境変数がJSバンドルに直接埋め込まれ、
  ブラウザのDevToolsから誰でも閲覧できてしまう。
  サーバーサイドだけで使うAPIキーには絶対に付けない。
- **YouTube Data API v3 Search.list**:
  - エンドポイント: `GET https://www.googleapis.com/youtube/v3/search`
  - 必須パラメータ: `part=snippet`, `q=<検索語>`
  - 結果: `items[].id.videoId`, `items[].snippet.title`, `items[].snippet.thumbnails`
  - クォータ: 100 calls/day、1リクエスト = 1 unit
- **YouTube IFrame Player API**:
  - JSライブラリを読み込むだけ（APIキー不要）
  - `onYouTubeIframeAPIReady()` で YT.Player を生成
  - `onStateChange` イベントで再生状態（playing/paused/ended）を検知
  - `getCurrentTime()` で現在の再生位置（秒）を取得可能

**理解深化：**
- **APIキー管理のセキュリティは「誰に見られるか」より「持続性」が重要**:
  会話内のAIに見られるのは一過性だが、JSバンドルに焼き付けると
  リポジトリの歴史に残り続け、訪問者全員が抽出可能になる。
  練習プロジェクトでも習慣として正しい管理を身につける。

  `useYouTubePlayer` フックや `YoutubeSearch` など完成度の高いコードが
  すでに含まれていた。前回のセッションで書かれたもの。
- **CONTEXT.md は実態と乖離しやすい**:
  技術スタック・移行状況・進捗など、書きっぱなしにせず
  実コードと照らし合わせて定期的にメンテナンスする必要がある。

### 💡 Key Learnings
**Newly Acquired:**
- **API Route Handler proxy pattern**: Browser → self-hosted API route → third-party API. Keeps API keys server-side and out of client bundles:
  ```tsx
  const apiKey = process.env.YOUTUBE_API_KEY;
  const url = new URL("https://www.googleapis.com/youtube/v3/search");
  url.searchParams.set("key", apiKey);
  ```
- **`NEXT_PUBLIC_` prefix meaning**: Vars with this prefix get inlined into JS bundles at build time, visible to anyone via DevTools. Never use for API keys that should stay server-side.
- **YouTube Data API v3 Search.list**: `GET /youtube/v3/search` with `part=snippet`, `q=<query>`, returns `videoId`, `title`, `thumbnails`. Quota: 100 calls/day, 1 unit each.
- **YouTube IFrame Player API**: No API key needed. Load JS library, create `YT.Player` in `onYouTubeIframeAPIReady()`, track state via `onStateChange`, get position via `getCurrentTime()`.

**Deeper Understanding:**
- **API key security is about persistence, not visibility**: AI seeing a key in conversation is ephemeral; a key baked into JS bundle persists in git history and is extractable by any visitor.
- **Draft branches persist across sessions**: The `draft/youtube-integration` branch already had high-quality code from a previous session.
- **CONTEXT.md drifts from reality**: Must regularly cross-check with actual code and update.
