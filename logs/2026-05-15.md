- **`filter` メソッドによる除外**: `session.id !== id` という条件を指定することで、削除対象以外の要素だけで構成された新しい配列を作成する手法。

**理解深化：**
- **Reactの不変性（Immutability）**: 既存の `sessions` 配列を直接書き換えるのではなく、`filter` が返す「新しい配列」を `setSessions` に渡すことで、Reactが正しく変更を検知し再描画を走らせる仕組み。

### 🔗 実装コード

```tsx
  const handleDelete = (id: string) => {
    // IDが一致しないものだけを残した新しい配列を作成して状態を更新
    setSessions(sessions.filter((session) => session.id !== id));
  };
```
