### 💡 学びのポイント
**新規習得：**
- **`useMemo` の正しい理解**: パフォーマンス最適化のためのフック。依存配列の値が変わったときだけ計算を再実行し、結果をキャッシュする。`useState` とは全く別の役割。
- **`"use client"` の正しい意味**: Next.js App Router ではデフォルトで Server Component。`"use client"` をつけると Client Component になり、`useState` や `useEffect` などブラウザAPIが使えるようになる。「web操作できるようにする」は不正確で、正確には「サーバーではなくブラウザで実行するコンポーネントですよ」という宣言。

**理解深化：**
- **「なぜ」を説明できないと理解したことにならない**: AIのレビューで、コメントが「何をしているか」の説明で止まっていて「なぜそう書くか」に触れていない部分を指摘された。コードの動作説明と設計意図の説明は別物。復習（STEP 6）では「なぜ」まで含めて書けるかを試すことになる。
- **AI添削の価値**: 自分では「わかってるつもり」だった間違いが、第三者（AI）の視点で明確になる。特に `useMemo` の誤解は、自分では気づけなかった盲点。

**トラブルシューティング：**
- **STEP 6への不安**: コメントを全部消して記憶だけで書き直すのは自信がない。あまりに理解が浅い部分があると実感したので、まずはP0のバグを直してから、焦らずSTEP 6に進むのが良さそう。

- **"Why" vs "What"**: AI review highlighted that many comments only described "what the code does" without explaining "why it's written this way." These are fundamentally different levels of understanding.
