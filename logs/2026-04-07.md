**理解深化：**
- **ReactのStateとUIの連動**: `useState` で管理されている `squares` 配列の状態が変更されると、コンポーネントが再レンダリングされ、`calculateWinner` 関数が再実行される。その結果が `status` 変数に反映され、画面の表示が変わる、という一連の流れをデバッグを通じて体感できた。
- **コンポーネントの親子関係**: 親コンポーネント(`Board`)が子コンポーネント(`Square`)の状態を管理し、コールバック関数(`onSquareClick`)を通じて子のイベントを親が処理する、という基本的なデータフローの理解が深まった。

**トラブルシューティング：**
- **課題**: 三目並べでいずれかのプレイヤーが勝利条件を満たしても、「Winner」が表示されない。
- **原因**: `calculateWinner` 関数内の `if` 文の条件式に誤りがあった。` (squares[b] === squares[a]) === squares[c]` という記述では、真偽値 (true/false) と文字列 ('X'/'O') を比較してしまい、意図した判定になっていなかった。
- **解決策**: 条件式を `squares[a] && squares[a] === squares[b] && squares[a] === squares[c]` に修正。これにより、「マスが空でない」かつ「3つのマスがすべて同じ値である」という正しい勝利条件を判定できるようになった。

```javascript
// 修正した勝者判定ロジック
function calculateWinner(squares) {
  const lines = [
    [0, 1, 2],
    [3, 4, 5],
    [6, 7, 8],
    [0, 3, 6],
    [1, 4, 7],
    [2, 5, 8],
    [0, 4, 8],
    [2, 4, 6],
  ];

  for (let i = 0; i < lines.length; i++) {
    const [a, b, c] = lines[i];
    // マスが埋まっており、かつ3つのマスが全て同じ値かを確認する
    if (squares[a] && squares[a] === squares[b] && squares[a] === squares[c]) {
      return squares[a];
    }
  }
  return null;
}
```
