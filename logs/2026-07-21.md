### 💡 学びのポイント
**新規習得：**
- **Clerk の keyless mode**:
  - 環境変数なしでも開発用の一時キーで認証が動く
  - 本番ではダッシュボードから本キーを発行する必要あり
- **Supabase RLS と Clerk の非互換性**:
  - Supabase RLS の `current_setting('app.user_id', true)` は Supabase Auth 用
  - Clerk を使う場合、RLS ポリシーは別の方式（もしくはアプリ側で制御）が必要
- **`NEXT_PUBLIC_` の typo に注意**:
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY` を `NEXT_PUBLIC_SUPABSE_ANON_KEY` と typo しており、`undefined` になりエラー
  - 環境変数名の typo はエラーメッセージがわかりにくいので注意

**理解深化：**
- **Supabase クライアントはモジュールレベルで即時実行しない**:
  - 初期段階で `createClient()` を呼ぶと env 未設定で落ちる
  - 関数でラップして遅延初期化 + シングルトンにするパターン
- **Next.js 16 の middleware 非推奨**:
  - `middleware.ts` → `proxy.ts` への移行が推奨されている
  - Clerk の `createRouteMatcher` も非推奨 → 各ページで auth チェックする方式に

**トラブルシューティング：**
- **Supabase 401 エラー（RLS ブロック）**:
  - RLS ポリシーが Clerk auth に対応していないためデータ操作が拒否された
  - 今は `alter table sessions disable row level security;` で回避
- **`aggregateByDate is not defined`（GrassCalendar 既存バグ）**:
  - 関数名 `aggregateByData` を `aggregateByDate` と typo
  - 他にも `mothLabels`/`lastMonthh`/`font-nomal` など多数の typo を一括修正

### 🔗 実装コード
- リポジトリ：`routine-tracker`（`draft/clerk-supabase-migration` ブランチ）
- 主要コード：
```tsx
// 遅延初期化＋シングルトンの Supabase クライアント
import { createClient } from "@supabase/supabase-js";

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL;
const supabaseAnonKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY;

let _client: ReturnType<typeof createClient> | null = null;

export function getSupabase() {
  if (_client) return _client;
  if (!supabaseUrl || !supabaseAnonKey) {
    throw new Error("Supabase の環境変数が設定されていません。");
  }
  _client = createClient(supabaseUrl, supabaseAnonKey);
  return _client;
}
```

### 💡 Key Learnings
**Newly Acquired:**
- **Clerk keyless mode**:
  - Development auth works without env vars using temporary keys
  - Need to claim real keys from dashboard before production
- **Supabase RLS incompatibility with Clerk**:
  - `current_setting('app.user_id', true)` only works with Supabase Auth
  - Clerk requires a different RLS approach (or app-level access control)
- **Watch for `NEXT_PUBLIC_` typos in env vars**:
  - `NEXT_PUBLIC_SUPABSE_ANON_KEY` typo (missing 'A') caused silent `undefined`
  - Env var typos produce confusing error messages

**Deeper Understanding:**
- **Don't eagerly initialize Supabase client at module level**:
  - `createClient()` at module level crashes if env vars are missing
  - Wrap in lazy function + singleton pattern instead
- **Next.js 16 middleware deprecation**:
  - `middleware.ts` → `proxy.ts` migration recommended
  - Clerk's `createRouteMatcher` also deprecated in favor of per-page auth checks

**Troubleshooting:**
- **Supabase 401 (RLS blocking access)**:
  - RLS policies didn't match Clerk auth, so data operations were rejected
  - Workaround: `alter table sessions disable row level security;`
- **`aggregateByDate is not defined` (pre-existing GrassCalendar bug)**:
  - Function name typo: `aggregateByData` called as `aggregateByDate`
  - Fixed 13 typos including `mothLabels`/`lastMonthh`/`font-nomal`
