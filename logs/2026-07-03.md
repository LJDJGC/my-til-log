### 💡 学びのポイント
**新規習得：**
- **`?.`（オプショナルチェーン）**: `obj?.prop` で obj が null/undefined なら undefined を返す。if ガードなしで安全にプロパティアクセスできる。
  - 例: `dayData?.byType[type]` → dayData が null でもエラーにならない
- **`??`（null合体演算子）**: `value ?? fallback` は value が null/undefined のときだけ fallback を使う。
- **`||` との違い**: `||` は falsy 全般（0, "", false）を置き換える。`??` は null/undefined だけ。

**理解深化：**
- **getLevel の完成形に `?.` / `??` を適用**:
  ```tsx
  const value = dayData?.byType[type] ?? 0;
  if (value === 0) return 0;
  if (value <= 40) return 1;
  if (value <= 60) return 2;
  if (value <= 80) return 3;
  return 4;
  ```
  1行で null/undefined ガードを書けるようになった。
- **STEP 6 の目的再確認**: 「本番マージ」ではなく「理解の定着」。閾値の正確さにこだわるより、関数の存在理由と入出力を説明できることが大事。
- **転職スケジュール**: 7月はカジュアル面談の時期。完璧を目指さず、練習として面談を受けるのが戦略。

**トラブルシューティング：**
- **`!expr ?? 0` の罠**: `!` は必ず boolean を返すので `?? 0` が機能しない。`?.` と `??` を正しく組み合わせるには `dayData?.byType[type] ?? 0` のように、`?.` の結果（undefined になりうる）に直接 `??` を繋ぐ。

### 💡 Key Learnings
**Newly Acquired:**
- **`?.` (Optional Chaining)**: `obj?.prop` returns undefined if obj is null/undefined. Safe property access without an explicit if guard.
- **`??` (Nullish Coalescing)**: `value ?? fallback` uses fallback only when value is null or undefined.
- **Difference from `||`**: `||` replaces all falsy values (0, "", false). `??` only replaces null/undefined.

**Deeper Understanding:**
- **Applying `?.` / `??` to getLevel**: One line handles both null dayData and undefined byType[type]:
  ```tsx
  const value = dayData?.byType[type] ?? 0;
  ```
- **STEP 6 Goal Confirmation**: The purpose is understanding retention, not production merge. Accuracy of thresholds matters less than being able to explain what the function does and why it exists.
- **Career Timeline**: July is for casual interviews. Don't wait for perfection — use early interviews as practice.

**Troubleshooting:**
- **The `!expr ?? 0` trap**: `!` always returns a boolean, so `?? 0` never fires. Correct pattern: `dayData?.byType[type] ?? 0` — chain `??` directly after `?.` (which can return undefined).
