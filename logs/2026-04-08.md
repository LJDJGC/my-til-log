### 💡 学びのポイント
**新規習得：**
- **Viteのセキュリティ仕様（DNSリバインディング対策）**：外部ホスト（CodeSandboxのURL）からのプレビューアクセスを許可するために、`vite.config.ts` の `server.allowedHosts` を設定する必要があるという、フロントエンドのインフラ設定の基礎を習得した。
- **配列からUIへの変換（`map`）**：JavaScriptの `Array.prototype.map()` を使用して、`history`（過去の手番の配列）のデータを Reactの `<button>` 要素（UI）の配列に変換し、画面にリスト表示する手法を実装した。

**理解深化：**
- **動的リストにおける問題の発見**：`map`でリストをレンダリングした結果、コンソールに `Warning: Each child in a list should have a unique "key" prop.` という警告が出た。ReactがDOMの変更を効率的に検知するために、一意の識別子（`key`）が必要であることを体感として理解した（解決は次回）。

**トラブルシューティング：**
- **課題**：CodeSandbox上でViteのプレビュー画面が「Blocked request」となり表示されない。
- **解決方法**：エラーメッセージの指示に従い、`vite.config.ts` にアクセス元のURLを許可する設定（`server.allowedHosts`）を追記することで解決。エラー文を読み解き、構成ファイルを書き換える対応力が身についた。

// history配列をmapでループし、過去の着手へ戻るボタンのリストを生成する
const moves = history.map((squares, move) => {
  let description;
  if (move > 0) {
    description = 'Go to move #' + move;
  } else {
    description = 'Go to game start';
  }
  return (
    // ※現在はkeyプロパティがないためコンソールエラーが発生している状態

    </li>
  );
});
