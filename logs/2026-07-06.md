### 💡 学びのポイント
**新規習得：**
- **`new Date("2026-07-06")` はどこでパースしているのか**: JavaScript エンジン（V8等）に組み込まれた機能。ISO 8601 形式（YYYY-MM-DD）の文字列を自動で解釈する。どこかのファイルで定義されているわけではなく、言語仕様として決まっている。
- **`getMonth()` は「数字」を返す**: 1月=0, 2月=1, ..., 12月=11。月名（"Jan"等）を返すわけではない。数字→月名の変換は自分で書く必要がある。
  ```tsx
  const MONTHS = ["Jan","Feb","Mar","Apr","May","Jun",
                  "Jul","Aug","Sep","Oct","Nov","Dec"];
  const month = date.getMonth(); // 7月→6
  MONTHS[month]; // → "Jul"
  ```

**理解深化：**
- **「書けない」と「わからない」の違い**: 前に教えてもらったことをそのまま書くのは学習の第一歩。大事なのは「書いたコードの意味を説明できるか」。
- **2日休んでも戻ってこれればOK**: 休むのも戦略。再開できた時点で勝ち。

### 💡 Key Learnings
**Newly Acquired:**
- **Where `new Date("2026-07-06")` parsing happens**: Built into the JavaScript engine (V8 etc.). It automatically interprets ISO 8601 format strings (YYYY-MM-DD) according to the language spec. No external config or file defines this behavior.
- **`getMonth()` returns a number, not a name**: January=0, February=1, ..., December=11. You must write the conversion from number to month name yourself using an array:
  ```tsx
  const MONTHS = ["Jan","Feb","Mar","Apr","May","Jun",
                  "Jul","Aug","Sep","Oct","Nov","Dec"];
  date.getMonth(); // July→6
  MONTHS[6]; // → "Jul"
  ```

**Deeper Understanding:**
- **The difference between "can't write" and "don't understand"**: Copying what you were taught is the first step. What matters is whether you can explain the meaning of the code you wrote.
