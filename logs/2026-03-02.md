Deepening Understanding:
- **How ​​to Use Design Patterns:**
- Since only one `Gameboard` is required within the app, the **Module Pattern (IIFE)** was adopted.
- Since multiple `Player`s need to be generated, the **Factory Function** was adopted.
- **Separation of Concerns:** The importance of managing board data (`board`) and game progress (`activePlayer`, etc.) as separate objects.
