### 💡 学びのポイント
**新規習得：**
- **JavaScript `filter` 関数の活用**: 配列から特定の要素を除外する際、破壊的な `splice` ではなく、新しい配列を返す `filter` を使うことがReactの不変性（イミュータビリティ）維持に最適であることを学んだ。
- **インライン関数の注意点**: `onClick={handleDelete(id)}` と書くと即時実行されてしまうため、`onClick={() => handleDelete(id)}` のようにアロー関数でラップして「予約」する必要があるという、初心者が陥りやすい罠を理解した。

**理解深化：**
- **Reactの再レンダリングトリガー**: 配列の中身を書き換えるのではなく、新しい配列の参照を `setState` に渡すことで初めてReactが変更を検知するという原則の再確認。
- **アクセシビリティ（A11y）**: アイコンのみのボタンには `aria-label` を付与し、スクリーンリーダー利用者に意図を伝える重要性。

**トラブルシューティング：**
- **レイアウトの調整**: `flex items-center gap-4` を使って、テキスト情報と削除ボタンを水平に等間隔で並べる手法。

### 💡 Key Learnings
**Newly Acquired:**
- **Utilizing JavaScript's `filter` function**: Learned that when removing specific elements from an array, using `filter` (which returns a new array) instead of the destructive `splice` is best for maintaining React's immutability principle.
- **Handling Inline Functions in React**: Understood the "trap" where writing `onClick={handleDelete(id)}` triggers the function immediately. Learned to wrap it in an arrow function like `onClick={() => handleDelete(id)}` to "schedule" the execution.

**Deeper Understanding:**
- **React Re-rendering Triggers**: Reconfirmed the principle that React only detects changes when a reference to a *new* array/object is passed to `setState`, rather than modifying the existing one in place.
- **Accessibility (A11y)**: The importance of adding `aria-label` to icon-only buttons to convey their purpose to screen reader users.

**Troubleshooting:**
- **Layout Adjustments**: Techniques for horizontally aligning text information and a delete button with consistent spacing using `flex items-center gap-4`.
