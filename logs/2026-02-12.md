### üí° Learning Points
**New Learnings:**
- **Module Pattern:** A design pattern that uses an IIFE (Immediately Invoked Function Expression) to group related methods and variables into a single object.
- **IIFE (Immediately Invoked Function Expression):** A technique that uses the `(function() { ... })();` syntax to create a scope that is executed immediately upon definition and inaccessible from outside.

Deepened Understanding:
- **Encapsulation:** Understand how closures are used to hide variables within a function (private) and allow them to be accessed only through the returned method (public API). This is the underlying concept of Factory Functions and the Module Pattern.

**Troubleshooting/Insights:**
- "Why do we need to make functions immediate?" ‚Üí I wrote some code and understood that the purpose is to create "singleton"-like behavior that doesn't pollute the global scope and is executed only once.
