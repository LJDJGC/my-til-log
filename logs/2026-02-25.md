### 💡 学びのポイント
**新規習得：**
- **Inside-out Development:** UI（HTML/CSS）からではなく、コアロジック（JSオブジェクト）から設計・実装する手法。
- **Immutability (不変性):** `getBoard` メソッドにおいて `return board` ではなく `return [...board]` とすることで、外部からの意図しないデータ改変を防ぐ設計。

**理解深化：**
- **Module vs Factory:** 
    - `Gameboard` はゲーム内に1つしか存在しないため **Module Pattern (IIFE)** を採用。
    - `Player` は複数生成されるため **Factory Function** を採用する使い分け。
- **Separation of Concerns (関心の分離):** データの保持（Gameboard）とゲーム進行（GameController）を分ける設計思想。

**トラブルシューティング：**
- 特になし（設計段階での考慮によりスムーズに進行）。

### 🔗 実装コード
- リポジトリ：https://github.com/LJDJGC/tictactoe-project-top
- 主要コード：
```javascript
const Gameboard = (() => {
    let board = ["", "", "", "", "", "", "", "", "", ];

    // 外部からの直接変更を防ぐため、コピーを返す（守りの設計）
    const getBoard = () => [...board];

    const placeMarker = (index, marker) => {
        if (board[index] === "") {
            board[index] = marker;
            return true;
        }
        return false;
    };

    return { getBoard, placeMarker};
})();
```
