### 💡 Key Learning Points
**New Learnings:**
- **IIFE (Immediately Invoked Function) and Module Pattern:** I learned how to execute a function immediately upon definition and hide its internal variables from the outside world (encapsulation) using the `(function() { ... })();` syntax.
- **Singleton (Single Instance):** I learned that IIFEs are suitable for objects that should only exist once within an app, such as a game board or a referee.

Deepening Understanding:
- **Difference between Factory Functions and IIFEs:** I learned the clear rule that "Factory Functions" are used for things like players that require multiple copies, and "IIFEs" are used for things that only require one copy.
- **Building a House from the Inside Out:** I learned design thinking that starts with building the "logic (internals)" that runs on the console, rather than creating the UI (appearance) first.

**Troubleshooting:**
- **Resolving the "Why IIFE and Not a Regular Function?" Question:** I was able to clarify that an IIFE is necessary to protect data from being arbitrarily rewritten from the outside, and as a constraint to prevent accidentally creating a second one and introducing bugs.
