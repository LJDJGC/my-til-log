### 💡 学びのポイント
**新規習得：**
- **IIFE（即時実行関数）とモジュールパターン：** `(function() { ... })();` という書き方で、定義と同時に実行し、内部変数を外部から隠蔽（カプセル化）する方法を学んだ。
- **シングルトン（1つだけのインスタンス）：** ゲームボードや審判役のように、アプリ内で1つしか存在してはいけないオブジェクトにはIIFEが適していることを知った。

**理解深化：**
- **Factory関数とIIFEの使い分け：** プレイヤーのように複数作るものは「Factory関数」、1つだけで良いものは「IIFE」という明確な基準を理解した。
- **Building a house from the inside out：** UI（見た目）を先に作るのではなく、まずコンソール上で動く「ロジック（内部）」から作り始める設計思考を学んだ。

**トラブルシューティング：**
- **「なぜ普通の関数ではなくIIFEなのか？」の解消：** 外部から勝手にデータを書き換えられないように保護するため、また誤って2つ目を作ってバグを生ませないための制約としてIIFEが必要であることを整理できた。

const gameboard = (function() {

  const getBoard = () => board;
  return { getBoard };

  return { name, marker };
