### 💡 学びのポイント
**新規習得：**
- **2次元配列の走査：** 配列の中に配列が入っている構造 (`[[0,1,2], ...]`) を `for` ループで処理する方法。
- **勝利条件のパターン化：** 盤面のインデックスを使って、勝ちパターンをデータとして定義する手法。

**理解深化：**
- **論理積 (`&&`) の活用：** 「AかつBかつC」という条件式を使って、3つのマークが揃っているかを判定するロジック。
- **バグ防止の重要性：** `board[a] !== ""` (空文字チェック) を入れないと、未入力のマス同士が等しいと判定されてしまう問題への対処。

**トラブルシューティング：**
- **課題：** `winningConditions` のループ処理をどう書けばよいか分からなかった。
- **解決：** `for` ループでインデックス `i` を回し、`winningConditions[i]` で個別の勝ちパターン配列を取り出し、それをさらに展開して `board` 配列のインデックスとして使う流れを理解・実装した。

### 🔗 実装コード

- 主要コード：勝利判定ロジック
```javascript
const checkWinner = () => {
    const board = Gameboard.getBoard();
    // 勝ちパターン8通りを走査
    for (let i = 0; i < winningConditions.length; i++) {
        const [a, b, c] = winningConditions[i]; // 分割代入で見やすく
        // 1. 空ではない 2. aとbが同じ 3. aとcが同じ
        if (board[a] && board[a] === board[b] && board[a] === board[c]) {
            return true;
        }
    }
    return false;
};
```
