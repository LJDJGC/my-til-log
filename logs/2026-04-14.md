### 💡 学びのポイント
**新規習得：**
- **依存配列（Dependency Array）の制御：** 第二引数が「未指定（毎レンダリング時）」「空配列 `[]`（マウント時のみ）」「変数指定 `[count]`（値の変更時）」で、副作用や関数の再生成タイミングがどう変わるかを理解した。
- **クリーンアップ関数のライフサイクル：** アンマウント時だけでなく、**依存変数が変化してエフェクトが再実行される直前**にも呼ばれることを確認した。
- **条件付きレンダリング：** JSX内では `if` 文が使えないため、三項演算子（`condition ? true : false`）や論理否定（`!isShow`）を用いてUIの表示/非表示を切り替える手法を実装した。
- JavaScriptにおける `Truthy` / `Falsy` の概念（`null`, `undefined`, `0` はFalsyとして扱われる点）の確認。

**理解深化：**
- **パフォーマンス最適化の重要性：** 大規模なアプリケーションになるほど、不要な再レンダリングや関数の再生成を防ぐ（`useCallback` などの活用）ことが、中長期的なパフォーマンス向上に直結する。

**トラブルシューティング：**
- `input` 要素の `value` 属性に state を直接紐付けるとリードオンリーになり入力できなくなる。`onChange` イベントハンドラを設定し、イベントオブジェクト（`e.target.value`）から値を取得して state を更新するフローを正しく実装した。

```tsx
// ※JS環境の解説をTypeScript環境に翻訳して実装
import { useState, useCallback, ChangeEvent } from "react";

export default function Home() {
  // 1. 型推論を効かせてStateを定義
  const [text, setText] = useState<string>("");
  const[isShow, setIsShow] = useState<boolean>(true);

  // 2. イベントの型（ChangeEvent<HTMLInputElement>）を明記し、文字数制限とトリム処理を実装
  const handleChange = useCallback((e: ChangeEvent<HTMLInputElement>) => {
    if (e.target.value.length > 5) {
      alert("5文字以内にしてください");
      return;
    }
    setText(e.target.value.trim()); // 前後の空白を自動除去
  },[]);

  // 3. 真偽値の反転（直前のStateに依存する場合は関数アップデートを利用する）
  const handleDisplay = useCallback(() => {
    setIsShow((prev) => !prev);
  },[]);

  return (
    <div className="container">
      {/* 三項演算子を用いた表示テキストの動的切り替え */}
      <button onClick={handleDisplay}>{isShow ? "非表示" : "表示"}</button>
      
      {/* 真偽値に応じたコンポーネントのマウント/アンマウント */}
      {isShow ? <h1>{text}</h1> : null}
      
      <input type="text" value={text} onChange={handleChange} />
    </div>
  );
}
```

**Newly Acquired:**
- **Controlling Dependency Arrays:** Understood how side effects and function regeneration timing change depending on whether the second argument is "unspecified (every rendering)", "empty array `[]` (only at mount)", or "variable specified `[count]` (when the value changes)".
- **Lifecycle of the Cleanup Function:** Confirmed that it is called not only when unmounting, but also **just before the effect is re-executed due to a change in the dependent variable**.

- **Conditional Rendering:** Since `if` statements cannot be used within JSX, a technique was implemented to toggle the display/hide of UI elements using the ternary operator (`condition ? true : false`) and logical negation (`!isShow`).

- Confirmation of the `Truthy`/`Falsy` concepts in JavaScript (that `null`, `undefined`, and `0` are treated as Falsy).

**Deepening Understanding:**
- **Importance of Performance Optimization:** In large-scale applications, preventing unnecessary re-rendering and function regeneration (using `useCallback`, etc.) directly contributes to long-term performance improvements.

**Troubleshooting:**
- Directly linking the `value` attribute of an `input` element to the state results in a read-only input field. A correct flow was implemented where an `onChange` event handler is set, the value is retrieved from the event object (`e.target.value`), and the state is updated.
