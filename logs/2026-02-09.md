- **Constructor vs Factory：** `this` や `prototype` の複雑さを避けたい場合、Factoryの方が可読性と安全性が高い。

### 💡 Key Learning Points
**New Learnings:**
- **Closure:** A combination of a function and the lexical environment in which it is declared. This allows the function to maintain access to variables in the outer scope.
- **Factory Functions:** A function that creates objects without using the `new` keyword. This avoids prototype pollution and allows for simpler encapsulation.

**Deepened Understanding:**
- **Encapsulation (Realizing Private Variables):** A method of using closures to create "true private variables" in JS that cannot be directly modified from outside (variable definition within a factory function).
- **Constructor vs. Factory:** If you want to avoid the complexity of `this` and `prototype`, Factory is more readable and safer.
