### 💡 学びのポイント
**新規習得：**
- **VS CodeのGUIによるGit操作:** `git add` や `git commit` をターミナルで打たずとも、ソース管理パネルからワンクリックで実行できる手法。また、変更箇所を赤緑のハイライトで視覚的に確認する「差分（Diff）チェック」の重要性を学んだ。
- **Props（プロップス）の概念:** 親コンポーネントから子コンポーネントへ渡す「おつかいのメモ書き（引数）」。Reactの単方向データフロー（上から下へ）の根幹であり、子は受け取ったProps（データや関数）を読み取り専用として扱う。

**理解深化：**
- **関数の「参照渡し」と「即時実行」の違い（超重要）:**
  `onClick={handleClick}` は「クリックされたらこの関数を呼んでね」という**関数そのもの（参照）**を渡している。
  一方、`onClick={handleClick(0)}` とカッコをつけてしまうと、画面が描画された瞬間に**その場で関数が実行されてしまい、Stateが更新されて再描画→また実行…という「無限ループ（Too many re-rendersエラー）」**を引き起こす。
- **クロージャとReactの関係:** Vanilla JSのIIFEで学んだ「関数が生まれた時のスコープ（親の変数）を記憶する仕組み（クロージャ）」が、Reactの `useState` やイベントハンドラ（親の変数を子が操作できる理由）の土台になっていることを腹落ちさせた。

**トラブルシューティング：**
- **事象:** `Square` コンポーネントに `handleClick(0)` のように引数付きの関数を直接渡すと、クリックする前に実行されて無限ループになる。
- **解決策:** `() => handleClick(0)` のように**アロー関数で包む**ことで、「クリックされた時に初めて `handleClick(0)` を実行する」という新しい関数を作り、それをPropsとして渡すことで無限ループを回避する手法（アロー関数による遅延実行）を理解した。

### 🔗 実装コード

### 💡 Learning Points
**Newly Acquired:**
- **Git operations using VS Code's GUI:** A method that allows you to execute `git add` and `git commit` with one click from the source control panel without typing them in the terminal. Also learned the importance of "diff checking," which visually confirms changes with red and green highlighting.
- **Concept of Props:** "Notes (arguments)" passed from parent component to child component. This is the core of React's unidirectional data flow (top to bottom), where children treat the received Props (data and functions) as read-only.

**Deepening Understanding:**
- **The Difference Between "Pass by Reference" and "Immediate Execution" of Functions (Extremely Important):**
`onClick={handleClick}` passes the **function itself (reference)**, meaning "call this function when clicked."
On the other hand, if you put parentheses around it, like `onClick={handleClick(0)}`, the function is executed immediately the moment the screen is rendered, causing an infinite loop ("Too many re-renders error") where the State is updated, re-rendering, and then execution occurs again...
- **The Relationship Between Closures and React:** I now fully understand that the mechanism for remembering the scope (parent variables) at the time a function was created (closures), which I learned in Vanilla JS's IIFE, forms the foundation of React's `useState` and event handlers (the reason why children can manipulate parent variables).

**Troubleshooting:**
- **Issue:** When passing a function with arguments directly to the `Square` component, such as `handleClick(0)`, it executes before the click, resulting in an infinite loop.
- **Solution:** I learned how to avoid the infinite loop by wrapping the function in an **arrow function**, like `() => handleClick(0)`, creating a new function that executes `handleClick(0)` only when clicked, and passing this as a Props. (This utilizes delayed execution via arrow functions.)
