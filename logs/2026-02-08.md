### ðŸ’¡ Learning Points
**New Learnings:**
- **Factory Functions:** A method for creating objects without using `new`. This reduces the risk of being confused by `this` behavior and allows for intuitive object return.
- **Closure:** A mechanism for "remembering" the scope in which a function was defined. This can be used to implement "private variables" that cannot be overridden from outside.
- **Module Pattern:** A method for using IIFEs (Immediately Invoked Functions) to create "singleton" objects (such as settings or calculator functions) that are unique to the entire application.

Deeper Understanding:
- Scope hierarchy: Global scope (anywhere), function scope (within functions), block scope (within `{}`, only `let/const`). Through Wes Bos's explanation, I rediscovered why `var` is deprecated in modern JS.
- Inheritance vs. composition: Factory functions use `Object.assign` to create new objects by "combining" only the necessary functionality.

Troubleshooting:
- When I tried to access a private variable and received `undefined`, I was able to confirm at the code level that this was intended encapsulation and not a bug.
