### 💡 学びのポイント
**新規習得：**
- **`TYPE_COLORS` の型定義**: `Record<string, string[]>` でタイプごとに5段階（0〜4）の色階調を保持。各レベルに Tailwind のクラス `bg-{color}-{shade}` を直接格納する設計パターン。dark mode対応で配列サイズが2倍になる。
  ```tsx
  const TYPE_COLORS: Record<string, string[]> = {
      study: [
          "bg-blue-50 dark:bg-blue-950",   // level 0
          "bg-blue-200 dark:bg-blue-800",  // level 1
          "bg-blue-400 dark:bg-blue-600",  // level 2
          "bg-blue-800 dark:bg-blue-200",  // level 3
          // ← 本来なら level 4 も必要（参考: draft/grass-visualization）
      ],
  };
  ```
- **写経時に気づいた typo**: 参考コードでは `bg-gren-*` になっていた。typoをそのまま写経してしまう罠。次回の検証ステップで発見・修正予定。

**理解深化：**
- **写経の覚悟（愚直に一行ずつ）**: 学習管理の儀式としての写経。丸写しではなく「なぜこのデータ構造か」「level 4がなぜないか」「dark:bg-*との関係は」と問いながら進める重要性を再確認。
- **CONTEXT.md の再設計力**: 最初の設計（外部視聴履歴取得）がAPI制約で不可能と判明した後、「アプリ内で完結させる」という代替案に素早くピポットした。これは単なる仕様変更ではなく、プロダクト価値の再定義＝ポートフォリオの質を上げる設計判断。

**トラブルシューティング：**
- **朝の起床失敗（5時目標達成できず）**: 「絶対に5時に起きる」と決めても、睡眠時間・体調によって達成できない日がある。
- **対策**: 「できなかった」と落ち込むのではなく、昼休みの学習時間を確実に確保するという現実的な代替ルートを実行。一貫性より継続性を優先するマインドセットが重要。

### 💡 Key Learnings
**Newly Acquired:**
- **`Record<string, string[]>` Color Schema**: A clean pattern where each activity type maps to an array of Tailwind classes representing intensity levels (0 to 4). Dark mode doubles the array entries (`bg-blue-50 dark:bg-blue-950`). This in-template design keeps styling fixed rather than computed.
- **Transcription Reveals Typo in Reference Code**: Noticed `bg-gren-*` instead of `bg-green-*` during manual typing — a spelling bug that would likely be missed with copy-paste. Catching these during transcription is the exact value of the handson approach.

**Deeper Understanding:**
- **The Ritual of Hand-Typing Code**: Each line demands a conscious "why is this structured this way?" question. For `TYPE_COLORS`, the questions are: why `Record<string, string[]>` and not a nested object? Why embed Tailwind classes instead of CSS variables? Why are there only 4 levels for workout but 4+(originally 5) for study?
- **Pivot as a Design Strength**: When the original plan (external YouTube history import) hit a hard API wall, the pivot to an "in-app search and auto-track" model wasn't just a fallback — it increased the product's self-contained value. This demonstrates real senior-level tradeoff navigation.

**Troubleshooting:**
- **Failed Morning Wake-up (5 AM target)**: Despite strong commitment, sleep schedule made the target unattainable today.
- **Countermeasure**: Instead of spiraling into guilt, executed the backup plan — protecting the lunch break study session. Consistency over intensity; the priority is that no day goes completely dark, not that every day is perfect.
