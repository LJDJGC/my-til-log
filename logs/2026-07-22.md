### 💡 学びのポイント
**新規習得：**
- **`useMemo` の使いどころ**:
  - 計算コストが高い処理（日付集計・色計算）をキャッシュする
  - 第2引数の配列が変わった時だけ再計算される
  - `[sessions]` → sessions が変わった時だけ再実行

**理解深化：**
- **Firestore 接続ブロックの実態**:
  - 広告ブロッカーは `firestore.googleapis.com` への WebSocket 通信を遮断する
  - エラーにならずに「待ち続ける」（Promise が解決しない）
  - → `await` すると後続処理が永久に実行されない
  - 対策: localStorage を先に書き、Firestore は fire-and-forget（待たない）
- **Firebase の初期化パターン**:
  - `getApps().length > 0 ? getApp() : initializeApp(config)` で二重初期化防止
  - モジュールレベルで即時実行せず、関数でラップして遅延初期化するパターンもある
- **期日から逆算した技術選定**:
  - Clerk + Supabase より Firebase 継続を選んだ判断
  - 「より良い技術」より「動いているものを壊さない」リスク管理

**トラブルシューティング：**
- **`ERR_BLOCKED_BY_CLIENT`（Chrome 拡張機能）**:
  - uBlock Origin などの広告ブロッカーが Firestore をブロック
  - localhost 開発環境でのみ発生（本番は自ドメイン経由なので問題なし）
  - 根本解決: Firestore 書き込みを `await` せず fire-and-forget に変更
- **Supabase RLS と Clerk の非互換性**:
  - `current_setting('app.user_id', true)` は Supabase Auth 専用
  - Clerk では別方式が必要 → 今回は Firebase 継続で解決

### 💡 Key Learnings
**Newly Acquired:**
- **When to use `useMemo`**:
  - Caches expensive computations (date aggregation, color calculation)
  - Only recalculates when the dependency array changes
  - `[sessions]` → recalculates only when sessions changes

**Deeper Understanding:**
- **How ad blockers affect Firestore**:
  - Ad blockers block WebSocket connections to `firestore.googleapis.com`
  - The connection doesn't error — it hangs forever
  - Solution: write to localStorage first, Firestore as fire-and-forget
- **Firebase initialization pattern**:
  - `getApps().length > 0 ? getApp() : initializeApp(config)` prevents double init
  - Can also wrap in lazy function pattern instead of module-level execution
- **Risk management in tech decisions**:
  - Firebase continuation over Clerk + Supabase was a schedule-driven decision
  - "Don't break what works" over "use better technology"

**Troubleshooting:**
- **`ERR_BLOCKED_BY_CLIENT`**:
  - Caused by uBlock Origin blocking Firestore connections
  - Only happens on localhost (production uses same origin)
  - Fix: fire-and-forget Firestore writes, localStorage first
- **Supabase RLS + Clerk incompatibility**:
  - `current_setting('app.user_id', true)` is Supabase Auth only
  - Clerk needs a different approach → resolved by sticking with Firebase
