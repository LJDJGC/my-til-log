### 💡 学びのポイント
**新規習得：**
- **Next.jsの学習順序:** モダンな自社開発企業を目指す上でNext.jsは必須だが、基礎となるReact（状態管理やコンポーネント設計）を理解する前に触れるとエラーの切り分けができず挫折に繋がる。必ず「React単体」→「Next.js」の順で学ぶ。
- **Just-in-Time学習の極意:** 公式ドキュメント（React.devなど）の長大な目次は「教科書」として上から順に読むのではなく、実装中に不明点が出た際やAIコードをレビューする際にピンポイントで引く「辞書」として扱う。

**理解深化：**
- **「文法を知らないまま作る」ことの真価:** 新しい言語やフレームワークを学ぶ際、文法を丸暗記してから開発を始める（学校型学習）のではなく、チュートリアルで手を動かしながら「ここでこの文法が必要なのか」と体感ベースで学ぶ（実践型学習）方が、エンジニアとしての習熟速度とROIが圧倒的に高いことの腹落ち。
- **JSの基礎体力の価値:** Vanilla JSで苦労して実装した「モジュール分割」や「クラスのカプセル化」の経験が、Reactのコンポーネント設計を学ぶための最強の準備運動になっていること。

**トラブルシューティング：**
- **事象:** 「Reactの基本文法すら知らない状態で、いきなり公式チュートリアル（三目並べ）に突入して大丈夫か」という学習アプローチに対する根源的な不安。
- **解決策:** React公式チュートリアル自体が「未経験者向け」に文法と実装を同時に学べるよう設計されていること、そして自分にはすでにJSの確固たる基礎（モジュールやthisの概念）があるため、チュートリアルを通じて「JSでの複雑な処理がReactでいかにシンプルに書けるか」というアハ体験を得ることが最速の学習法であると理解し、不安を払拭した。

### 💡 Key Learning Points
**New Acquisition:**
- **Next.js Learning Order:** Next.js is essential for aiming to become a modern in-house development company, but learning it before understanding the foundational React (state management and component design) will lead to frustration due to the inability to isolate errors. Always learn "React alone" → "Next.js". - **The Secret to Just-in-Time Learning:** The lengthy table of contents of official documentation (such as React.dev) should not be read sequentially as a "textbook," but rather used as a "dictionary" to pinpoint specific points when questions arise during implementation or when reviewing AI code.

**Deepening Understanding:**
- **The True Value of "Building Without Knowing the Grammar":** When learning a new language or framework, it's far more effective for engineers to learn by doing through tutorials and experiencing "why this grammar is needed here" (practical learning) rather than memorizing the grammar before starting development (school-style learning).

- **The Value of a Strong Foundation in JS:** The experience of struggling to implement "module splitting" and "class encapsulation" in vanilla JS serves as the best preparation for learning React component design.

**Troubleshooting:**
- **Issue:** A fundamental concern about the learning approach: "Is it okay to jump straight into the official tutorial (Tic-Tac-Toe) without even knowing the basic syntax of React?"
- **Solution:** I realized that the official React tutorial itself is designed for beginners, allowing them to learn syntax and implementation simultaneously. Furthermore, because I already have a solid foundation in JavaScript (modules and the concept of `this`), I understood that the fastest way to learn was to gain the "aha!" moment through the tutorial—seeing how complex JavaScript processes can be written simply in React—and thus dispelled my anxiety.
