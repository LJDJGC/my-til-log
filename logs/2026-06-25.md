### 💡 学びのポイント
**新規習得：**
- **git filter-repo による履歴改変**: `--replace-text` オプションで全コミットの特定文字列を一括置換できる。force push で GitHub の履歴も上書き可能。ただし GitHub のキャッシュには旧データが残る可能性があるため、完全に消したい場合はサポートにパージ依頼が必要。
- **git rm --cached の使い方**: ファイルをディスクに残したまま Git の追跡だけ外す。`.gitignore` と組み合わせることで「ローカルでは使うが GitHub には上げない」ファイルを実現できる。

**理解深化：**
- **写経 typo 修正の効果**: 先週の TIL で「発見済みの typo を一掃する」と書いていたが、今日実際に修正した。typo だらけのコードがコンパイルエラーを出し、一つずつ直すプロセスで変数名・関数名の一貫性がなぜ重要なのかを体感できた。特に `dataKey` と `dateKey` の混在は、コンパイルは通っても論理的におかしい挙動を引き起こす。TypeScript の型チェックだけでは防げない「命名の不整合」に対する注意力が養われた。
- **CONTEXT.md の設計ミス**: 個人情報をプロジェクトのコンテキストファイルに書いてしまったのは明らかな設計ミス。学習の進捭や設計判断を記録する場と、個人のキャリア情報を記録する場は分けるべき。今後はキャリア情報は別ファイル or プライベートノートに分離する。

**トラブルシューティング：**
- **個人情報が GitHub に露出**: 過去のコミットに会社名・年収・年齢・転職先候補が含まれていた。`git filter-repo` で履歴を書き換え、force push で上書き。さらに CONTEXT.md 自体を Git 管理から外して再発防止。

### 💡 Key Learnings
**Newly Acquired:**
- **git filter-repo for History Rewriting**: The `--replace-text` option can batch-replace specific strings across all commits. Force-push overwrites GitHub history too. However, GitHub may still cache old data — contact support for complete purge if needed.
- **git rm --cached Pattern**: Remove a file from Git tracking while keeping it on disk. Combined with `.gitignore`, creates a "local-only" file pattern perfect for private configuration files.

**Deeper Understanding:**
- **Why Naming Consistency Matters**: The `dataKey` vs `dateKey` mix would compile fine (both are valid identifiers) but cause logical bugs at runtime. TypeScript can't catch this. Hand-fixing these typos builds an intuition for naming discipline that linters alone cannot teach.
- **CONTEXT.md Design Mistake**: Mixing project context with personal career info in the same file was poor design. Future separation: project decisions in CONTEXT.md, career/personal info in a separate private note.

**Troubleshooting:**
- **Personal Data Exposed on GitHub**: Company name, salary, age, and job target companies were in git history across multiple commits. Solution: `git filter-repo` to rewrite history + force push + remove file from tracking.
- **Cross-Branch Application**: After filter-repo, cherry-picking the privatization commit to all 13 branches caused modify/delete conflicts on each. Resolved consistently with `git rm --cached CONTEXT.md` on each branch.
