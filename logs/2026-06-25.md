### 💡 学びのポイント
**新規習得：**
- **git filter-repo による履歴改変**: `--replace-text` オプションで全コミットの特定文字列を一括置換できる。force push で GitHub の履歴も上書き可能。ただし GitHub のキャッシュには旧データが残る可能性があるため、完全に消したい場合はサポートにパージ依頼が必要。
- **git rm --cached の使い方**: ファイルをディスクに残したまま Git の追跡だけ外す。`.gitignore` と組み合わせることで「ローカルでは使うが GitHub には上げない」ファイルを実現できる。

**理解深化：**
- **写経 typo 修正の効果**: 先週の TIL で「発見済みの typo を一掃する」と書いていたが、今日実際に修正した。typo だらけのコードがコンパイルエラーを出し、一つずつ直すプロセスで変数名・関数名の一貫性がなぜ重要なのかを体感できた。特に `dataKey` と `dateKey` の混在は、コンパイルは通っても論理的におかしい挙動を引き起こす。TypeScript の型チェックだけでは防げない「命名の不整合」に対する注意力が養われた。
- **CONTEXT.md の設計ミス**: 個人情報をプロジェクトのコンテキストファイルに書いてしまったのは明らかな設計ミス。学習の進捭や設計判断を記録する場と、個人のキャリア情報を記録する場は分けるべき。今後はキャリア情報は別ファイル or プライベートノートに分離する。

**トラブルシューティング：**
- **個人情報が GitHub に露出**: 過去のコミットに会社名・年収・年齢・転職先候補が含まれていた。`git filter-repo` で履歴を書き換え、force push で上書き。さらに CONTEXT.md 自体を Git 管理から外して再発防止。
- **全ブランチへの適用**: filter-repo 後、全13ブランチに同じ変更を cherry-pick するのに手間取った。各ブランチで CONTEXT.md の内容が微妙に異なるため modify/delete コンフリクトが毎回発生。`git rm --cached CONTEXT.md` で一律解決できた。
