### 💡 学びのポイント
**新規習得：**
- **`{[0,1,2,3,4].map(...)}` によるレベルの視覚化**: レベル0〜4の数値配列を `.map()` で回して、各レベルに対応する色の正方形を `<span>` でレンダリング。配列リテラルを直接JSX内に埋め込むイディオム。
  ```tsx
  {[0, 1, 2, 3, 4].map((level) => (
      <span key={level} className={`inline-block h-3 w-3 rounded-sm ${DEFAULT_COLORS[level]}`} />
  ))}
  ```
- **凡例のUI設計**: 「少 □□□□□ 多」の配置で、色の濃さが活動量に対応することを直感的に伝える。`flex items-center gap-4` で横並び＋均等配置。

**理解深化：**
- **写経で気づく auto-import の罠**: 今回のコミットで `import { spawn } from "child_process"` が自動追加されている（6行目）。これは Next.js のブラウザ環境では使えない Node.js モジュールで、おそらくエディタの auto-import が誤爆したもの。写経中に「なぜこの import が？」と気づけるかどうかが理解のバロメーター。
- **CSSクラスの typo**: `w-3rounded-sm`（スペース欠落）→ Tailwind では無効なクラス名になり、四角が正しく表示されない。写経＋目視チェックの重要性。
