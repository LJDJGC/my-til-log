### 💡 学びのポイント
**新規習得：**
- **`useMemo` による集計値のメモ化パターン**: コンポーネント内で複数の `useMemo` をチェーンさせ、依存関係を明確にしながら計算結果をキャッシュする設計。
  ```tsx
  // ① sessions → dayMap & maxDayMinutes を集計
  const { dayMap, maxDayMinutes } = useMemo(() => aggregateAndMax(sessions), [sessions]);
  // ② weeks → 日付配列 を生成
  const days = useMemo(() => generateDateRange(weeks), [weeks]);
  // ③ days → 週ごとに分割
  const weeksList = useMemo(() => chunkIntoWeeks(days), [days]);
  ```
  各 `useMemo` の依存配列が前の計算結果を受け取るチェーン構造になっている。

- **`getLevel` の比率ベース段階分け**: 0〜4の5段階で、最大値に対する比率によって強度を決定。`if (ratio <= 0.1) return 0` で「完全にゼロに近い」を弾く設計。0.1刻みではなく不均衡な閾値（0.1 / 0.25 / 0.5 / 0.75）なのは、低強度の日が大量に発生するのを防ぐため。

- **`Map<string, DayData>` での日付単位集計**: `Map.has()` + `Map.get()` で既存日付をチェックしながら集計。最終ループで `Object.entries()` を回して最も多いタイプを `dominantType` に設定する2パス方式。

### 💡 Key Learnings
**Newly Acquired:**
- **Chained `useMemo` Pattern**: Multiple `useMemo` hooks linked by dependency arrays — sessions → dayMap+maxMinutes → days → weeksList → monthLabels. Each hook depends on the previous one's output, creating a clean data pipeline.
- **`getLevel` Ratio-Based Intensity Scaling**: Five levels (0-4) based on percentage of the max daily value. Thresholds (0.1/0.25/0.5/0.75) are intentionally unbalanced — the steep early cutoffs prevent too many days from appearing at low intensity.
- **`Map<string, DayData>` Two-Pass Aggregation**: First pass records total and per-type duration per date. Second pass iterates with `Object.entries()` to determine `dominantType` by finding the max value across activity types.

**Deeper Understanding:**
- **Transcription as Typo Magnet**: Hand-typing reveals errors that copy-paste would silently swallow:
  - `DayData.data` → should be `date` (breaks downstream date lookups)
  - `aggregateByData` → should be `aggregateByDate` (function name + internal ref mismatch)
  - `mothLabels` / `lastMonthh` → missing 'n' / extra 'h'
  - `bg-gren-*` → missing 'e' in "green"
  - `GrassCalender` (filename) → missing 'a'

  Each typo fixes a conceptual gap. Catching them is the whole point of this learning method.

**Troubleshooting:**
- **Learning Time Fragmentation Strategy**: Morning wake-up continues to be inconsistent. But having backup slots (lunch, evening) means no day goes dark. This multi-slot strategy effectively de-risks the single-point-of-failure that was "morning only."
