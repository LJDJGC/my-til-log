### üí° Learning Points
**New Learning:**
- **Closures:** A combination of a function and the lexical environment in which it is declared. This mechanism allows access to the function's internal variables even after the function has finished executing.
- **IIFE (Immediately Invoked Function Expression):** A function that is executed immediately upon definition. A common technique for creating a private space without polluting the global scope.

Deepening Understanding:
- **Encapsulation:** The Module Pattern hides variables (internal state) that should not be directly overwritten from the outside, and explains the importance of "information protection."
- **Namespace Protection:** A design concept that avoids the risk of increasing global variables (directly under `window`) by consolidating them into a single object (module).

**Troubleshooting:**
- **Explaining "Why closures are necessary":** Rather than simply writing "working code," we logically explained, using MDN examples (such as `makeAdder`), why variables are retained in memory and how this helps "maintain state."
