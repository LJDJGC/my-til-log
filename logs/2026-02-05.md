### üí° Key Learning Points
**New Learnings:**
- **HTML Data Attributes (`dataset`)**: A method for safely embedding JS data in HTML elements as "markers."
- **`crypto.randomUUID()`**: A modern method for generating unique IDs with an extremely low probability of duplication.

**Deeper Understanding:**
- **Data Immutability**: Using array indexes (0, 1, 2, etc.) as IDs creates the "debt" of mismatching numbers upon deletion, but this is not a concern with UUIDs.
- **Single Source of Truth**: The idea is that if the array (data) is manipulated correctly, consistency can be maintained simply by redrawing the screen (display) based on that data.

**Troubleshooting:**
- To address the concern that "when pressing the delete button, it becomes difficult to know which book to delete," we found a solution by adding an ID (dataset) to the HTML and using that as a key to `filter` the array.
