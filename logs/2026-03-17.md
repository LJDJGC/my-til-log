### 💡 学びのポイント
**新規習得 & 理解深化（知識チェックより）：**
- **コンストラクターとクラスの違い：** `class` は従来のコンストラクター関数の「シンタックスシュガー（書きやすくした構文）」。ただし、クラスは `new` なしで呼ぶとエラーになるなど、より厳格で安全な設計になっている。
- **ゲッターとセッター (`get` / `set`)：** プロパティ（変数の値）を直接書き換えさせず、取得や変更の際に「バリデーション（チェック）」などの処理を挟むための機能。
- **継承 (`extends` / `super`)：** 親クラスの機能を子クラスが引き継ぐ仕組み。`super()` を呼ぶことで、親の初期化処理を使い回せる。
- **プライベートクラス機能 (`#`)：** プロパティ名の先頭に `#` をつけることで、クラスの外から絶対にアクセス（変更・取得）できないようにする安全装置（カプセル化）。
- **静的プロパティ (`static`)：** （今朝の実証通り）インスタンスごとではなく、クラス（設計図）そのものに紐づく便利なユーティリティ機能。

### 💡 Learning Points
**New Acquisition & Deeper Understanding (from Knowledge Check):**
- **Difference between Constructors and Classes:** `class` is "syntax sugar" for the traditional constructor function. However, classes have a stricter and safer design; calling them without `new` will result in an error.

- **Getters and Setters (`get` / `set`):** Functions that prevent direct modification of properties (variable values) by inserting validation (checks) during retrieval and modification.

- **Inheritance (`extends` / `super`):** A mechanism where a child class inherits the functionality of its parent class. Calling `super()` allows the parent's initialization process to be reused.

- **Private Class Functionality (`#`):** A safety mechanism (encapsulation) that prevents access (modification/retrieval) from outside the class by prefixing property names with `#`.

- **Static Properties (`static`):** (As demonstrated this morning) A convenient utility function tied to the class (blueprint) itself, rather than per instance.
