### üí° Key Learning Points
**New Learnings:**
- **The Role of Prototypes:** By defining methods in `prototype` instead of writing them directly in the constructor, functions are shared across all instances, reducing memory consumption (optimizing memory efficiency).
- **crypto.randomUUID():** A practical ID generation method that generates unpredictable unique identifiers and prevents bugs caused by data duplication and index misalignment.

**Deeper Understanding:**
- **Separation of Data and DOM:** The basis of a "data-driven" approach: re-executing `displayBook()` (drawing) after manipulating the `myLibrary` array (data).
- **Dataset Property:** A method for safely embedding JavaScript-accessible information in HTML elements using `data-*` attributes.

**Troubleshooting:**
- **Breaking textContent Specification:** I encountered an issue where executing `card.textContent = "..."` deleted all child elements (such as buttons) within that element. I found a solution that preserved the UI structure by creating a separate text-only element (such as a `p` tag) and using `appendChild`.
