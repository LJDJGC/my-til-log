```jsx
// Reactに依存しない純粋なJavaScriptのヘルパー関数として外に切り出す
function calculateWinner(squares) {
  const lines = [
    [0, 1, 2],[3, 4, 5], [6, 7, 8], // 横
    [0, 3, 6],[1, 4, 7], [2, 5, 8], // 縦[0, 4, 8], [2, 4, 6]             // 斜め
  ];
  for (let i = 0; i < lines.length; i++) {
    const [a, b, c] = lines[i];
    if (squares[a] && squares[a] === squares[b] && squares[a] === squares[c]) {
      return squares[a];
    }
  }
  return null;
}
```

### 💡 学びのポイント
**新規習得：**
- **CSS Modules:** Next.jsに標準搭載されているCSSの管理手法。`[name].module.css` というファイル名にしてインポートすることで、クラス名にランダムな文字列が自動付与される。これにより「名前の衝突」を完全に防ぐことができ、BEMなどの面倒な命名規則から解放される。
- **Linkコンポーネント (`<Link>`) :** Next.jsにおける高速な画面遷移の要。従来の `<a>` タグでは画面全体がリロード（再読み込み）されて遅くなるが、`<Link>` を使うと裏側で次のページのデータを先読み（prefetch）してくれるため、一瞬で画面が切り替わる。

**理解深化：**
- **コンポーネント指向の真髄:** 画面（HTML）をコンポーネント単位で分割するだけでなく、「CSSもコンポーネントごとに分割して管理する」というReact/Next.jsの美しい設計思想が腹落ちした。
- **プロのコミット＆デプロイ:** 動画内で、意味のある単位で細かくコミットを分け、GitHubにPushした瞬間にVercel（サーバー）に自動でデプロイ（公開）される一連の流れを見て、自分が今設定している環境がいかに実務に近いかを実感できた。

**トラブルシューティング：**
- 特になし（インプット中心のため）。ただし、「`pages` フォルダ」の解説については、自分の環境（App Router）では `src/app` に置き換えて解釈するという脳内翻訳を問題なく行えている。
